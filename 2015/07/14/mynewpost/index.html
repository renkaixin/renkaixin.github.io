Git 版本控制流程

在使用Git进行版本控制，定义开发生命周期中的每个阶段分别对应的branch

  • master HEAD代码与稳定的生产环境代码同步
  • develop HEAD代码代表了下一版本release的最新代码更改
  • feature 新功能开发分支
  • release 版本发布分支
  • hotfix 线上问题修改分支

各分支详解

branch type branch off from merge back into
master main - -
develop main - -
feature supporting develop develop
release supporting develop develop & master
hotfix supporting master develop & master

feature branches - 新功能分支

新功能分支是branch off from develop，它的生命周期伴随着开发某一特定新功能，新功能开发完成后，将feature分支merge到develop中
